資料倉儲 Data Warehouse
資料倉儲概述 Data Warehouse Overview （一）
Sep 15, 2025

資料倉儲 Data Warehouse

　　資料倉儲 是一種資料儲存的概念，日常生活中各種交易或是操作紀錄可能會先在應用系統上面被記錄下來，再送到資料倉儲去統一儲存，以便後續根據商業分析、BI、ML的需求能夠快速取得使用資料。引用Bill Inmon提出的資料倉儲特性大致如下：

整合性 Integrated : 資料倉儲整合來自不同系統的資料，透過清洗與轉換，保持資料的一致性跟正確性。

主題導向 Subject-Oriented : 不管資料來自哪些系統、以什麼樣的形式儲存，資料倉儲應根據業務主題去設計，而非著重在日常的交易處理。

時間變異性 Time Variant : 資料倉儲會儲存時間序列的資料（歷史資料），可根據時間的變化對資料進行追蹤和分析。

非暫存性Non-Volatile : 為了保持資料的穩定性以及歷史資料的完整性，資料幾乎保持在唯讀的狀態，若有修改的情況也會保留原始資料。


集中式 VS 分散式

開頭的示意圖中，將所有資料都送進單一的資料庫的形式稱作 集中式資料倉儲(Centralized Data Warehouse) 。這是理想情況下的預設架構，最傳統且簡易，也滿足One-Stop Shopping(OSS)，可以從此處取得所有所需的資料，供報表、BI、分析等等用途。

另有 分散式資料倉儲(Distributed Data Warehouse) 的概念，將資料儲存在不同主機系統或者是不同地區，使用上也有更高的難度。


資料市集 Data Mart

　　Data Warehouse的資料通常會提供給下游使用，根據不同的範疇，可能是部門、地區、功能等，去切分專用的區塊，稱為 資料市集(Data Mart) 。回顧文章開頭的示意圖，資料來源系統/客戶端應用程式提供交易資料，Data Warehouse整合資料，Data Mart取得整合過的資料進行商業分析或報表需求等等，因此Data Mart一般來說需仰賴Data Warehouse提供的資料，也因此Data Mart的資料規範較一致，Data Flow基本上就是從左往右的資料流。


相依 VS 獨立

　　上述Data Mart資料一律來自Data Warehouse的型態稱作 相依資料市集(Dependent Data Mart) ，另外還有 獨立資料市集(Independent Data Mart) 是直接跳過Data Warehouse從來源系統撈取資料，獨立運作，相當於自身就是較小的Data Warehouse。

　　Independent Data Mart在彼此之間資料規範較混亂，沒有一致性；且資料流之間沒有直接關聯，每個Data Mart都有不同的來源，下游在取得資料時也來自各個不同的Data Mart，如果將Data Flow畫出來通常都相當雜亂。


　　取用各種資料來源時並非單純寫入資料倉儲，不同的系統內對資料定義很容易有落差，在設計前必須要花大量時間先釐清每一份來源資料的內容定義，根據主題去統一的資料的階層、屬性、規範等等，這非常依賴對上游資料內容以及下游資料需求的理解。

　　舉例來說，不同系統的幣別及小數精確度可能不同、現金流是逐銀存認列還是逐票據認列，這些細節往往影響資料倉儲的設計，影響後續BI或OLAP Cubes分析的正確性。理想情況下，可以先從各種來源整合出一份 Master Data ，Master Data需要有高度的資料治理來維護，結合良好的Schema設計，作為後續建立維度表跟事實表最權威的Single Source of Truth(SSOT)。
